197/* Get the master session management client. This master client gets
198 a client id, that may be specified by the '--sm-client-id' command
199 line option. A master client will be generated by 'mate-init'.
200 If possible the master client will contact the session manager
201 after command-line parsing is finished (unless
202 'mate_client_disable_master_connection' was called). The master
203 client will also set the SM_CLIENT_ID property on the client leader
204 window of your application.
205
206 Additionally, the master client gets some static arguments set
207 automatically (see 'mate_client_add_static_arg' for static
208 arguments): 'mate_init' passes all the command line options which
209 are recognised by gtk as static arguments to the master client. */
210MateClient *mate_master_client (void);

212/* Get the config prefix for a client. This config prefix provides a
213 suitable place to store any details about the state of the client
214 which can not be described using the app's command line arguments (as
215 set in the restart command). You may push the returned value using
216 'mate_config_push_prefix' and read or write any values you require. */
217const gchar* mate_client_get_config_prefix (MateClient *client);

219/* Get the config prefix that will be returned by the previous function
220 for clients which have NOT been restarted or cloned (i.e. for clients
221 started by the user without `--sm-' options). This config prefix may be
222 used to write the user's preferred config for these "new" clients.
223
224 You could also use this prefix as a place to store and retrieve config
225 details that you wish to apply to ALL instances of the app. However,
226 this practice limits the users freedom to configure each instance in
227 a different way so it should be used with caution. */
228const gchar* mate_client_get_global_config_prefix (MateClient *client);
229
230/* Set the value used for the global config prefix. The config prefixes
231 returned by mate_client_get_config_prefix are formed by extending
232 this prefix with an unique identifier.
233
234 The global config prefix defaults to a name based on the name of
235 the executable. This function allows you to set it to a different
236 value. It should be called BEFORE retrieving the config prefix for
237 the first time. Later calls will be ignored.
238
239 For example, setting a global config prefix of "/app.d/session/"
240 would ensure that all your session save files or directories would
241 be gathered together into the app.d directory. */
242void mate_client_set_global_config_prefix (MateClient *client,
243 const gchar* prefix);

245/* Returns some flags, that give additional information about this
246 client. Right now, the following flags are supported:
247
248 - MATE_CLIENT_IS_CONNECTED: The client is connected to a session
249 manager (It's the same information like using
250 MATE_CLIENT_CONNECTED).
251
252 - MATE_CLIENT_RESTARTED: The client has been restarted, i. e. it
253 has been running with the same client id before.
254
255 - MATE_CLIENT_RESTORED: This flag is only used for the master
256 client. It indicates, that there may be a configuraion file from
257 which the clients state should be restored (using the
258 mate_client_get_config_prefix call). */

271/* The session manager usually only restarts clients which were running

273 the manager restart the client:

276 - never (MATE_RESTART_NEVER). */

280/* The mate-session manager includes an extension to the protocol which

282 a number of run levels. This function may be used to inform the
283 mate-session manager of where this client should appear in this
284 run level ordering. The priority runs from 0 (started first) to 99
285 (started last) and defaults to 50. Users may override the value
