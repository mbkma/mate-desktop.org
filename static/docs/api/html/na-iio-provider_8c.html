◆ na_iio_provider_item_changed()

void na_iio_provider_item_changed ( const NAIIOProvider *  instance)

na_iio_provider_item_changed: @instance: the calling NAIIOProvider.

Informs &prodname; that this NAIIOProvider @instance has detected a modification in (at least) one of its items (menu or action).

This function may be triggered for each and every NAObjectItem -derived modified object, and should, at least, be triggered once for a coherent set of updates.

When receiving this signal, the currently running program may just want to immediately reload the current list of items, menus and actions (this is for example what &caja; plugin does); it may also choose to ask the user if he is willing to reload such a current list (and this is the way &cact; has chosen to deal with this message).

Note that application NAPivot/NAUpdater pivot is typically the only object connected to this signal. It acts so as a filtering proxy, re-emitting its own 'items-changed' signal for a whole set of detected underlying modifications.

Since: 2.30
