102/*
103 * description of MateMDI signals:
104 *
105 * gint add_child(MateMDI *, MateMDIChild *)
106 * gint add_view(MateMDI *, GtkWidget *)
107 * are called before actually adding a mdi_child or a view to the MDI. if the handler returns
108 * TRUE, the action proceeds otherwise the mdi_child or view are not added.
109 *
110 * gint remove_child(MateMDI *, MateMDIChild *)
111 * gint remove_view(MateMDI *, GtkWidget *)
112 * are called before removing mdi_child or view. the handler should return true if the object
113 * is to be removed from MDI
114 *
115 * void child_changed(MateMDI *, MateMDIChild *)
116 * gets called each time when active child is changed with the second argument
117 * pointing to the old child. mdi->active_view and mdi->active_child still already
118 * hold the new values
119 *
120 * void view_changed(MateMDI *, GtkWidget *)
121 * is emitted whenever a view is changed, regardless of it being the view of the same child as
122 * the old view or not. the second argument points to the old view, mdi->active_view and
123 * mdi->active_child hold the new values. if the child has also been changed, this signal is
124 * emitted after the child_changed signal.
125 *
126 * void app_created(MateMDI *, MateApp *)
127 * is called with each newly created MateApp to allow the MDI user to customize it (add a
128 * statusbar, toolbars or menubar if the method with MateUIInfo templates is not sufficient,
129 * etc.).
130 * no contents may be set since MateMDI uses them for storing either a view of a child
131 * or a notebook
132 */

168/*
169 * the following two functions are here to make life easier if an application
170 * creates objects (like opening a window) that should "keep the application
171 * alive" even if there are no MDI windows open. any such windows should be
172 * registered with the MDI: as long as there is a window registered, the MDI
173 * will not destroy itself (even if the last of its windows is closed). on the
174 * other hand, closing the last MDI window when no objects are registered
175 * with the MDI will result in MDI being gtk_object_destroy()ed.
176 */
