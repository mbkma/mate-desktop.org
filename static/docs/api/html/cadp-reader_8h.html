cadp_reader_iimporter_import_from_uri: @instance: the NAIImporter provider. @parms: a #NAIImporterUriParms structure.

Imports an item.

Returns: the import operation code.

As soon as we have a valid .desktop file, we are most probably willing to successfully import an action or a menu of it.

GLib does not have any primitive to load a key file from an uri. So we have to load the file into memory, and then try to load the key file from the memory data.

Starting with N-A 3.2, we only honor the version 2 of NAIImporter interface, thus no more checking here against possible duplicate identifiers.
